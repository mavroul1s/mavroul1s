## `> cat prompt_engineering.md`

Designing structured language inputs to reliably elicit precise, high-quality outputs from large language models.

| Technique | Description |
|---|---|
| **System Prompt Design** | Defining model role, constraints, persona, and output format at the instruction layer |
| **Chain-of-Thought** | Forcing step-by-step intermediate reasoning before final output generation |
| **Few-Shot Prompting** | Embedding representative input-output pairs to anchor model behavior in-context |
| **Zero-Shot Prompting** | Task specification with no examples -- pure instruction and format control |
| **RAG Pipeline Design** | Structuring retrieval-augmented prompts to inject retrieved context efficiently |
| **Multi-Model Orchestration** | Chaining specialized models via prompt handoffs for complex multi-step pipelines |
| **Output Structuring** | Enforcing JSON, XML, markdown, or custom schemas via instruction and validation |
| **Adversarial Testing** | Stress-testing prompts for edge cases, instruction-following failures, and hallucination |

Applied across: research pipelines, data labelling automation, CV annotation workflows, and generative media projects.
